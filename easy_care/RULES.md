# 📜 EasyCare 코딩 및 협업 규칙 (v3.6)

## 1. UI/UX 및 테마 원칙
- **폰트/가독성:** 본문 최소 `18px`, 중요 수치 `bold` 및 `22px+` 적용. 터치 영역 최소 `60px`.
- **전역 다크모드:** 모든 페이지는 시스템 설정 혹은 전역 상태에 따라 다크모드가 즉각 반영되어야 함.
- **다국어 지원:** 시스템 설정을 따르되, 기본 언어는 **한글**로 고정함.
- **일관된 진단 로직:** 모든 스크린은 동일한 의학적 기준을 공유하며, 화면 간 상태 불일치는 절대 금지됨.

## 2. 지능형 OCR 입력 로직
- **사진 촬영:** 카메라 촬영 기능을 통해 input창에 자동 입력되어야 함. (모바일 전용)
- **추출 로직:**
  - **숫자 1개:** 해당 숫자를 `sugar`(혈당)로 자동 판단.
  - **숫자 2개 이상:** `혈당` 키워드 검색 후 인접한 숫자를 혈당 수치로 판단.
  - **숫자 3개 이상:** `혈압` 키워드가 있는 곳의 숫자를 수축기(큰 값)와 이완기(작은 값)로 판단.
- **재촬영:** 재촬영 버튼을 상시 노출하며, 재촬영 시 기존 필드를 초기화하고 로직을 재수행함.

## 3. 화면별 특화 규칙
### Dashboard
- 최근 혈당을 메인으로 하되, 혈압 수치를 하단/측면에 병기 (혈당 대비 60~70% 크기).
- **위험 알림:** 혈당, 혈압, 당화혈색소 중 하나라도 위험 수준(`isDanger`)일 경우 상단에 경고 배너를 노출하고 카드 색상을 강조함.

### StatsScreen (그래프) - [v1.7 업데이트]
- **차트 디자인:** 혈당/혈압 그래프 분리(Toggle). 전체 그래프 선 두께는 얇게(2px) 유지하고 데이터 점(Dot)을 강조.
- **기준선 (중앙 정렬):** 정상 범위 가로 점선(Baseline)이 항상 차트의 정중앙에 위치하도록 상하 여백을 동적으로 대칭 계산.
- **최고/최저 표시:** 차트 내에 최고치(빨간 점선)와 최저치(파란 점선)를 가로선으로 긋고 해당 수치를 명시.
- **터치 툴팁 & 인디케이터:** - 점 클릭 시 세로 인디케이터 선은 차트 선과 구분되는 **연한 회색 점선** 사용.
  - 툴팁 텍스트는 `혈당: 120` 또는 `수축기: 130` 형태로 값의 종류를 명확히 표기.
  - 해당 기간의 최고/최저점에는 툴팁(날짜+수치)을 항상 노출.
- **동적 요약 피드백:** 토글 버튼의 상태(혈당/혈압)에 따라 요약 텍스트와 테마 색상이 동적으로 변경되어야 함.

### StatsScreen (상세 리스트)
- **조건부 색상:** 수치 상태(정상/낮음/높음)에 따라 숫자 폰트 색상을 다르게 출력.
- **작성자 표시:** 가족 공유 기능에 따라 각 기록이 누구의 것인지(나 또는 가족 별칭) 명시함.
- **형식:** `상태(공복 등) | 메모: 내용` 구조로 출력 (구분자 `|` 사용).

### MyPage (마이페이지/설정) - [v1.8 업데이트]
- **정보 관리:** 사용자의 닉네임(수정 가능)과 계정 이메일을 상단에 명확히 표시.
- **테마 제어:** 전역 다크모드 전환 스위치를 제공하여 접근성 향상.
- **계정 정책:** 로그아웃 및 회원탈퇴 버튼을 배치. 탈퇴 시에는 데이터 영구 삭제에 대한 안내 및 재확인(Double Check) 팝업 필수.

## 4. 데이터 저장 및 환경
- **저장 필드:** `type`, `sugar`, `systolic`, `diastolic`, `hba1c`, `memo`, `timestamp`, `creatorCode` 필수 포함.
- **사용자 정보 관리:** 카카오 로그인 시 Firestore `users` 컬렉션에 프로필과 고유 `familyCode`를 저장하여 기기 변경 시에도 데이터를 보존함.
- **리포트:** 지정 기간 데이터를 **Excel 또는 PDF(CSV)** 형태로 다운로드 기능 구현. (모바일 전용)
  - **리포트 컬럼:** [일시, 구분, 혈당(BST), 혈압(BP), 당화혈색소(HbA1c), 메모, 작성자]
- **가족 연동:**
  - 가족 코드를 통해 연결된 사용자 간에는 서로의 건강 데이터(혈당, 혈압)를 읽기 전용으로 열람할 수 있어야 함.
  - 가족 등록 시 누구인지 식별하기 위한 '메모(별칭)'를 작성해야 하며, 연결 화면에서 수정 가능해야 함.
  - 가족 목록 최상단에는 '나 (본인)'를 고정 배치하여 통일된 내비게이션 경험을 제공함.
  - 가족 목록에서 특정 인물을 선택 시, 해당 인물의 데이터로 필터링된 통계 화면으로 즉시 연결되어야 함.
- **Auth:** 회원가입, 로그인, 로그아웃 등 인증 흐름을 명확히 관리해야 함. 로그아웃 시 세션 초기화 및 초기 진입점으로 강제 이동.
- **웹 세션 관리:** 웹 환경에서 계정 전환이 원활하도록 로그인 시 항상 `Prompt.login` 옵션을 적용함.
- **회원탈퇴:** 탈퇴 요청 시, 해당 사용자와 관련된 모든 개인 데이터(혈당/혈압 기록, 메모 등)는 **Firebase에서 완전히 삭제**되어야 함.
- **인프라:** Firebase Cloud Firestore (asia-northeast3).
- **표준 약어:** 앱 내 모든 레이블은 혈당(BST), 혈압(BP), 당화혈색소(HbA1c) 약어를 병기함.
- **빌드 및 의존성:** 패키지 간 의존성 충돌 방지를 위해 안드로이드의 `build.gradle.kts`(`desugar_jdk_libs` 등) 및 iOS의 `Podfile`(`GoogleMLKit/TextRecognitionKorean` 등)과 같은 네이티브 빌드 파일의 라이브러리 버전을 명시적으로 관리함.
- **분석 설정:** 불필요한 빌드 아티팩트 및 외부 패키지 예제 코드로 인한 노이즈 방지를 위해 `build/` 디렉토리는 `analysis_options.yaml`에서 분석 제외(`exclude`) 처리함.
- **웹(Web/Chrome) 개발 환경:**
  - 카카오 로그인 도메인 보안 정책 준수를 위해 웹 테스트 포트는 항상 **`8080`**으로 고정함 (`launch.json` 설정 필수).
  - 모바일 전용 기능(OCR, 알림)은 `kIsWeb` 가드를 통해 실행을 차단하여 런타임 오류를 방지함.
- **iOS 개발 환경:**
  - 특정 라이브러리의 호환성 이슈를 우회하기 위한 인위적인 아키텍처 제외 로직은 제거하고, 네이티브 `arm64` 빌드를 기본으로 함. 시뮬레이터 이슈 발생 시 실제 기기 테스트를 권장함.

## 5. 공통 디자인 시스템 및 코드 컨벤션
- **주요 색상 (Color Palette):**
  - **Primary (기본):** `Color(0xFF0052CC)` (테마 Seed, 정상 수치 기준 색상)
  - **배경색 (Background):** 라이트 모드 `Color(0xFFF8F9FE)`, 다크 모드 `Color(0xFF010813)`
  - **상태 색상 (Status Colors):**
    - **정상:** 혈당 `Color(0xFF0052CC)`, 혈압 `Color(0xFF047857)`, 당화혈색소 `Colors.purple`
    - **주의/경고:** `Colors.orange`, `Colors.orange.shade800` (저혈당, 공복혈당장애 등)
    - **위험 (Danger):** `Colors.red.shade800` (고혈당, 고혈압, 당뇨 수준)
  - **메뉴 카드 색상:** 기록 `Color(0xFF4338CA)`, 통계 `Color(0xFFB45309)`, 가족 `Color(0xFF047857)`, 마이페이지 `Color(0xFF374151)`
- **UI/UX 인터랙션 및 컴포넌트:**
  - **터치 피드백:** 카드나 버튼 터치 시 `AnimatedScale(scale: 0.95)`을 사용하여 시각적 눌림 효과 제공.
  - **모서리 곡률 (Border Radius):** 메인 대시보드 카드 `32px`, 메뉴 카드 `28px`, 기본 버튼 `16px`를 사용하여 부드러운 곡선 UI 유지.
  - **타이포그래피:** `displayLarge` 32px, `titleLarge` 22px, `bodyLarge` 18px 기준 준수. 아이콘 내부 원형 배경 적용으로 시인성 강화.
- **데이터 모델 및 아키텍처:**
  - **진단 로직 포맷:** 의학적 진단 반환값은 `Map<String, dynamic>` 타입으로 `{"label", "color", "msg"}` 형태를 통일하여 UI와 모델의 결합을 최소화.
  - **보안/환경 변수:** 카카오 SDK 등 API 키는 절대 소스코드 내 하드코딩하지 않고, 최우선적으로 `flutter_dotenv`를 활용하여 `.env`에서 불러오기 수행 (iOS의 경우 `Podfile` 빌드 스크립트를 통해 `.env`에서 파싱하여 `Info.plist`에 동적 주입).

## 6. 최신 API 컨벤션 (Flutter 3.27+)
- **Color 투명도:** `withOpacity`는 정밀도 손실 문제로 deprecated되었으므로, 반드시 `withValues(alpha: ...)`를 사용함.
- **데이터 공유:** `Share` 클래스의 정적 메서드 대신 `SharePlus.instance.share(ShareParams(...))` 구조를 지향함.
- **비동기 안전성:** 비동기 작업 후 `BuildContext`를 사용할 때는 반드시 `if (context.mounted)` (State 클래스 내부라면 `if (mounted)`) 체크를 수행하여 안전성을 확보함.
