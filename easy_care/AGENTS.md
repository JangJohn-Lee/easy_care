## 🚀 로드맵 및 진행 상태

### [완료 단계]
- **Step 1~4:** 기본 CRUD, 하이브리드 대시보드 및 통합 진단 엔진 구축 완료.
- **Step 5: 정밀 통계 및 리포팅 고도화 (완료)**
  - `fl_chart` 기반: 기준선 중앙 정렬 알고리즘, 최고/최저치 가로선 및 상시 툴팁 적용.
  - 차트 인디케이터 및 툴팁 UI/UX 어르신 친화적 개선 완료.
  - **대화형 툴팁 상세화**: 그래프 점 클릭 시 년/월/일/시/분, 측정 타입(공복 등), 메모를 줄바꿈하여 가독성 있게 노출.
  - **최고/최저점 날짜 라벨**: 그래프 내 최고/최저 수치에 해당 기록의 날짜(YY/MM/DD) 병기 완료.
  - 동적 요약 피드백(혈당/혈압 분리) 적용 완료.
  - 특정 기간 데이터를 Excel로 내보내는 리포트 추출 기능 완료 (`excel`, `share_plus` 패키지 활용).
  - 상세 로그: 캘린더 연동 리스트 UI 개선 및 **기록 수정 기능(Edit Mode)** 도입 완료.
- **Step 7: 소셜 연동 및 커뮤니티 (Final)**
  - 회원가입, 카카오 로그인, 마이페이지(닉네임 설정, 계정 관리, 가족 연결) 및 가족 초대 시스템 구축.
  - 가족 연결(`family_screen.dart`): 6자리 고유 초대 코드 자동 생성 및 클립보드 복사 기능 구현. 가족 코드 입력 시 식별용 메모(별칭) 작성 및 수정 기능.
  - 가족 간 데이터 공유: 연결된 가족의 혈당 및 혈압 기록 교차 조회(열람) 기능.
  - 삭제 시 'Double Check' 다이얼로그 및 `RULES.md` 공통 디자인 시스템 전면 반영 완료 (Primary Color `0xFF0052CC` 적용, 터치 최소 영역 `60px`, 폰트 `18px+`, `AnimatedScale(0.95)` 터치 피드백, 모서리 곡률 통일 등).
  - 스마트 푸시 알람(식후 2시간 등).
  - 마이페이지 내 회원탈퇴(개인 데이터 완전 삭제 포함) 기능 구현.
- **최신화 및 유지보수 (v3.7 ~ v4.0)**
  - 환경 변수(`.env`) 내 Kakao JS 키 통합 및 안드로이드 설정 동기화.
  - **대시보드-통계 연동**: 최근 기록 카드 클릭 시 해당 날짜의 캘린더 상세 내역으로 즉시 이동 기능 구현.
  - **데이터 필터링 개선**: `creatorCode`가 없는 레거시 데이터를 '나'의 기록으로 포함하도록 필터 로직 고도화.
  - **UI 안정성 확보**: `Expanded`, `Flexible`, `softWrap` 적용을 통한 가로 픽셀 오버플로우 전면 해결.

## 💡 핵심 확정 기능
1. **하이브리드 대시보드:** 혈당(메인)과 혈압(서브)을 한눈에 보는 직관적 UI 및 위험 상태 강조.
2. **멀티 차트 분석:** 혈당/혈압/당화혈색소의 기간별 추이 및 가족별 데이터 필터링 제공.
3. **통합 진단 엔진:** 앱 전체에 동일하게 적용되는 무결성 의학 알고리즘.

## 📁 lib 디렉토리 파일 구조 및 기능

### 1. 진입점 및 설정 (`lib/`)
- **`main.dart`**: 애플리케이션 진입점. Flutter 엔진, `.env` 환경 변수, 카카오 SDK, 알림 서비스 및 Firebase 초기화를 담당하며, 전역 테마(라이트/다크 모드) 전환 상태를 관리합니다. 웹/모바일 플랫폼 판별 로그가 포함되어 있습니다.

### 2. 모델 및 상태 로직 (`lib/models/`)
- **`health_stat.dart`**: `HealthRecord` 데이터 모델 정의. `sugar`, `systolic`, `diastolic`, `hba1c` 필드 포함. 개별 지표 진단 및 전체 위험 여부(`isDanger`) 판별 로직이 통합되어 있습니다.

### 3. 화면 UI (`lib/screens/`)
- **`dashboard_screen.dart`**: 메인 화면. 최근 기록 요약 및 위험 수치 발생 시 긴급 배너를 노출합니다. 텍스트 잘림 방지 레이아웃이 적용되어 있습니다.
- **`family_screen.dart`**: 가족 초대 및 연결 관리 화면. 서버 기반 영구 코드를 공유합니다. 통합 리스트에서 '나' 또는 가족을 클릭 시 통계 화면으로 자동 필터링 이동합니다.
- **`input_screen.dart`**: 기록 등록 화면. 수동/OCR 입력을 지원하며, 당화혈색소 입력을 포함합니다. 웹 환경에서는 OCR 기능이 비활성화됩니다.
- **`login_screen.dart`**: 카카오 로그인 화면. 로그인 시 Firestore `users` 테이블과 연동하여 고유 가족 코드를 동기화합니다.
- **`my_page_screen.dart`**: 사용자 설정 화면. 프로필 확인, 테마 토글, 로그아웃 및 데이터 영구 삭제를 동반하는 탈퇴 기능을 제공합니다.
- **`stats_screen.dart`**: 분석 리포트 화면. '전체/나/가족' 필터를 통해 데이터를 조회하며, 선택된 작성자 기준의 맞춤형 리포트 추출이 가능합니다.

### 4. 핵심 서비스 (`lib/services/`)
- **`notification_service.dart`**: 로컬 푸시 알림 엔진. 웹 환경에서는 `kIsWeb` 가드에 의해 작동이 중지됩니다.

### 5. 공통 위젯 (`lib/widgets/`)
- **`menu_card.dart`**: 대시보드용 카드 버튼 컴포넌트. 애니메이션 피드백과 어르신 친화적 규격을 준수합니다.

### 6. 안드로이드 플랫폼 설정 (`android/`)
- **`app/build.gradle.kts`**: 안드로이드 빌드 설정. `desugar_jdk_libs` 2.1.5 버전 및 한국어 OCR 모델 의존성을 관리합니다.

### 7. iOS 플랫폼 설정 (`ios/`)
- **`Runner/AppDelegate.swift`**: iOS 생명주기 및 알림 델리게이트 설정.
- **`Podfile`**: iOS 의존성 관리. 네이티브 `arm64` 빌드를 기본으로 하며 카카오 키 주입 스크립트를 포함합니다.
- **`Runner/Info.plist`**: 앱 권한 및 메타데이터 설정. `FLTEnableImpeller`를 `false`로 설정하여 렌더링 안정성을 확보했습니다.
